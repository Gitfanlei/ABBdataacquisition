Create Controller

A controller object is created in the following two ways. Creating a controller object is an entry point to accessing controller data.

  • Using Network Scanner, which scans the network and returns a list of controllers.
  • Using Controller Guid.

NetDumpRapid

This example explains the procedure to retrieve all routine names from the Rapid Module.

  1. All the tasks are retrieved from controller Rapid.
  2. All the modules are then retrieved from each Tasks from controller Rapid.
  3. Each Routine is then traced from the Module and printed out on the console.
    Figure 2:NetDumpRapid

VirtualController Events

This example explains the procedure to retrieve all routine names from the Rapid Module.

  1. A virtual Controller is retrieved using a network scanner.
  2. Subscribe the controller to the OperationModeChanged Event,
  3. A mode change on the Virtual FlexPendant, will trigger the changed mode event on the controller OperationModeChanged method.
